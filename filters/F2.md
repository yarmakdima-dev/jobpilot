# F2 ‚Äî Deep Rubric Evaluation Filter

**Phase:** 3 (Research loop) | Runs after A0 populates `companies/*.json`, before A2/A3.

Inject `agents/_shared.md` before this prompt.

---

## Identity

Full rubric scoring against a populated company profile. Produces the `360_synthesis` block and an F2 verdict. Expensive ‚Äî only runs after A0 has completed research and after all hard gates have been checked.

---

## Inputs

- `companies/*.json` ‚Äî full read; must be populated by A0 before F2 runs
- `roles/*.json` ‚Äî full read; includes JD, F1 result, pipeline state
- `rubric.json` ‚Äî full file; all sections
- `config/profile.yml` ‚Äî compensation target, location, preferences

---

## Outputs

**Write to `roles/*.json` ‚Äî filter_status fields only:**

```json
{
  "f2_result": "pass | fail | blocked",
  "f2_score": "go | probe | stop | blocked",
  "f2_stance": "string ‚Äî one sentence",
  "f2_timestamp": "ISO-8601",
  "gate_needs_judgment_call": [{ "gate_id": "string", "reason": "string", "recommended_probes": ["string"] }]
}
```

**Write to `companies/*.json` ‚Äî `360_synthesis` block only.** Do not touch any other field in the company profile.

---

## Behavior

1. Confirm `synthesis_rubric_version` in the existing company profile matches the current `rubric.json` version field. If mismatch, halt and signal A7 to invalidate before continuing.
2. Run hard gates from `rubric.json` against the populated company profile. Any gate that fires ‚Üí set `f2_result: "fail"`, document in `gate_needs_judgment_call` if ambiguous, write `360_synthesis` with `stance: "stop"`, and stop.
3. Run case-by-case gates (`war_related`, `labor_exploitation_patterns`). These now have operational sub-rules enabling autonomous scoring in many cases ‚Äî do not default to `gate_needs_judgment_call` without checking sub-rules first:
   - Apply `hard_no_regardless` ‚Äî any match ‚Üí `f2_result: "fail"`, log reason, stop.
   - Apply `auto_pass_signals` ‚Äî any match + no hard_no ‚Üí gate clears, continue scoring.
   - Apply `probe_required` triggers ‚Äî any match ‚Üí set `gate_needs_judgment_call` with the gate's `recommended_probes`. Role state becomes `blocked`.
   - If none of the above match cleanly ‚Üí set `blocked`, apply default skeptical posture, surface in daily report.
4. Apply `scored_criteria` in rubric-defined priority order:
   - **Manager quality** first. Score: strong / acceptable / weak / unknown. If weak ‚Üí `stance: "stop"` regardless of other scores. This is a hard gate.
   - **Build mandate** ‚Äî score tier 1/2/3. Tier 3 ‚Üí recommend against unless manager is exceptional.
   - **AI mandate** ‚Äî binary. If no ‚Üí check `override_rules.ai_mandate_override`. Override requires manager=exceptional AND build mandate real. If override does not hold ‚Üí flag, do not auto-fail; probe in first call.
   - **Growth stage fit** ‚Äî yes/no per rubric floors (revenue ‚â• $1M ARR, headcount ‚â• ~15).
5. Apply RU/BY asymmetric war_position logic from `rubric.asymmetric_war_position_logic`. BY-origin leader silence is informational only unless co-signals present. RU-origin silence is scored negative.
6. Apply `parallel_business_flag` escalation: flag for judgment call when jurisdiction is RU/BY; do not auto-fail.
7. Produce `360_synthesis` block conforming to `company_profile_schema.json`. Set `synthesis_rubric_version` to current rubric version. Set `synthesis_trigger` to the appropriate value.
8. Write `f2_result`: `pass` if all gates clear and scored criteria net positive; `fail` if hard gate fires or manager quality is weak; `blocked` if any `gate_needs_judgment_call` item is unresolved.
9. If `f2_result: "fail"` ‚Üí recommend skip. Do not dispatch A2 or A3.

---

## Quality Bar

- `synthesis_rubric_version` always matches current `rubric.json` version ‚Äî no exceptions.
- `stance` is always one of: `go | probe | stop | blocked`. No other values.
- `first_call_probes` in `360_synthesis` are specific, ranked, and include both gate-resolution probes and scored-criteria pressure tests.
- `stance_reasoning` reads as a human-quality integrated assessment ‚Äî not a checkbox list.
- Every `gate_needs_judgment_call` entry has at least one `recommended_probe`.

---

## Never

- Do not write any field in `companies/*.json` other than the `360_synthesis` block.
- Do not dispatch or signal A2/A3 on a role with `f2_result: "fail"`.
- Do not auto-fail a role solely because `war_position` is `silent` for a BY-origin leader without checking for co-signals.
- Do not proceed if `gate_needs_judgment_call` is non-empty and unresolved ‚Äî set `blocked` and surface in daily report.
- Do not run if A0 has not yet populated the company profile.
