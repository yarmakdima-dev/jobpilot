## Things worth revisiting in later sessions

### 1. "Results rule" is a deeper principle, not just an AI-mandate override

It surfaced in the AI-mandate discussion but the way Dima phrased it —
*"results rule, cause at the end of the day anything and everything will be effected by AI"* —
reads like a core operating principle. When Layer 3 (decision overrides) comes around, probe
whether "I can deliver results without being asked" is the override mechanism Dima uses in
OTHER judgment calls, not just the AI one. Likely yes. If so, it belongs elevated as a meta-rule.

### 2. Silence-as-signal logic

The Russian carve-out requires explicit public anti-war positioning from leadership; silence
is a negative signal. This is logically unusual — inverted default. Worth checking in Layer 3
whether Dima applies this pattern anywhere else (e.g., does silence on other values-relevant
topics also read as negative? Leadership style, DEI, labor, environmental position?).
Probably yes in some domains, no in others. Map it.

### 3. Case-by-case gates need sub-rules eventually

Two gates are currently "strong survey, case by case" with no operational sub-rules:
- War-related
- Labor-exploitation industries

These will need sub-rules before an agent can score them autonomously. Either extract in
a dedicated mini-session, or let them accumulate through the decision-log (Layer 3 artifact)
as real cases arrive.

### 4. Domain exclusion edge cases still pending

Flagged but not fully resolved:
- **Crypto / speculative finance** — fintech serving crypto-adjacent but non-speculative use
  cases. Needs a case when it comes up.
- **Gambling adjacencies** — fantasy sports, skill-games with prizes. Not resolved.
- **Dating adjacencies** — confirmed exclusion is romantic/sexual matching; "functions as
  prostitution-adjacent" is the broader filter. Professional/friendship/family-matching
  explicitly OK.

### 5. Collective C-suite as a concept

Dima framed a visionary CEO who delegates operational work as positive IF the C-suite
functions as a "collective CEO." This is an interesting lens and probably relevant for
Layer 2 (voice) — the way he thinks about teams and leadership structure likely shows up
in how he writes about them. Flag for cross-reference.

### 6. The hand-off test ("what will you stop doing?")

The sharpest operational test of Layer 1. Worth promoting in any agent prompt that scores
roles: this single question discriminates between healthy minutiae-experts and
control-oriented CEOs better than any other signal surfaced in this session.

### 7. Compensation structure flexibility

Floor is $3K/month net, absolute. Target is ~$200K/year total comp. The envelope between
those two numbers has room for different comp structures. In a later session (or in the
trust-map for the application agent), worth clarifying: does Dima want the agent to surface
roles across that whole range, or filter to something tighter by default?

### 8. What we did NOT cover in Layer 1

Things mentioned in the meta-prompt that weren't pushed on:
- No explicit weighting in numeric terms (percentage or score). Dima gave a ranking and
  a gating rule. Ranking is sufficient for most scoring purposes but a downstream agent
  may need rough numeric weights. Come back to this only if an agent stalls on it.
- False positives / false negatives as paired examples — we got positive/negative examples
  for the top criteria but didn't systematically build the "would pass the test but he'd
  still reject" column for every criterion. Layer 3 will likely generate these organically
  through real decision overrides.

## Process observations (for tuning future sessions)

- Dima processes verbally, as predicted. Answers come out dense, multi-clause, often folding
  3–4 distinct rules into one paragraph. Unpacking and playing back extracted rules worked
  well — he corrected and refined when playbacks were slightly off (e.g., the growth-stage
  correction opening up corporates).
- He responded well to pressure tests (counter-examples, edge cases). The "CEO who doesn't
  interview you deeply — false negative?" probe got the best data of the session.
- When given options (A/B/C), he picks cleanly. No agonizing. Good to remember for Layer 2
  and 3 where branching questions will be useful.
- Meta-check at the end ("keep going, stop, or lighter cleanup?") worked well. Default
  to offering a stopping point when a layer is complete.
